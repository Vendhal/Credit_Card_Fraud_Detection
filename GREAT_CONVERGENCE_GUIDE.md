## 🛡️ Anti-Overfitting Mechanisms (NEW!)

### 1. Formation Shuffling (Every 15 rounds)
```
🔄 Shuffling Formations:
├── Warriors mixed into NEW groups
├── Prevents static groupthink
├── Forces adaptation to new teammates
└── Maintains diversity
```

### 2. Individual Evaluation (Every 25 rounds)
```
📊 Evaluating Individual Performance:
├── Test each warrior ALONE (not in formation)
├── Detect "carried" warriors
├── If collective F1 >> individual F1:
│   └── Penalty: warrior.vijayas *= 0.8
└── Ensures everyone contributes!
```

### 3. Weighted Karma Distribution
```
🎯 WEIGHTED Karma (not equal!):
├── Calculate individual F1 scores
├── Guru gets 50% bonus + contribution bonus
├── Shishyas get base + contribution
└── Being carried = penalty!
```

**Why**: Prevents weak warriors from surviving on strong formation's coattails!

## 🧹 Clean Data - No Leakage! (NEW!)

### The Problem (FIXED!)
```
❌ OLD: Scaler fitted on all_train (train + test)
   = Data leakage!
   = F1 inflated by 1-2%

✅ NEW: Scaler fitted on train ONLY
   = No leakage!
   = Honest performance!
```

### Clean Preprocessing Process
```
Step 1: Split FIRST (80/20 stratified)
        ├── Train: 394 frauds + 227,451 normals
        └── Test: 98 frauds + 56,864 normals

Step 2: Fit MinMaxScaler on TRAIN ONLY
        └── Scaler learns from train data only!

Step 3: Transform BOTH with train-fitted scaler
        ├── Train data: normalized
        └── Test data: normalized (no influence!)

Step 4: Save clean data
        └── data/preprocessed_clean/
```
